    • Null is a value that you assign to a variable that means "Nothing".

      var x = null;

    • A JS Promise is essentially a callback that's used in partnership with asyncronous functions. Let's say you have a function that is fetching JSON from an API. You want to fire one response when the data returns and another if the data is not returned. Promises handle this for you.

  4. Difference between var, let, & const

    • Var

      • Can be redefined multiple times using the var keyword

        var width = 5;
console.log(width)

var width = 6;
console.log(width)

// Both will log. Neither throw an error.

      • Not block scoped. Instead, they are function scoped.

        function setWidth(){
  var width = 5;
  console.log(width)  
}

setWidth();  //Logs 5, because var 'width' exists within the fucntion

console.log(width);  //Throws an error. 'width' does not exist outside of the function scope

      • variable names that are declared with var are hoisted to the top of their containing function scope. There, they are given no assignment.

        For Example:

        
console.log(typeof name); // undefined
var name = "John";

/* The above two lines, after hoisting, really looks like this: */

var name;
console.log(typeof name): // undefined
var name = "John";



/* Const and let do something similar, only they
don't instantiate the variables, for some reason.
Instead of Undefined, you only get a refference 
error. */

console.log(typeof name); // ReferenceError
let name = "John";

console.log(typeof name); // ReferenceError
const name = "John";

    • Let

      • Block Scoped

      • Can't be declared twice in the same scope

      • CAN be updated

      • CAN be used in for loops when you want to assign the value of 'i' to an element in the dom, say.

        For Example:

        let arr = [];

for ( let 1 = 0; i < 10; i++ ){
	arr.push(`< div >${i}< /div >`);
}
console.log(arr); // < div >1, < div >2, etc...

    • Const

      • Block Scoped

      • Can't be declared twice in the same scope

      • Can't be updated.

        (This only means that the variable ITSELF can't be updated. If our const contains an object, we CAN updated the values inside of that object. The thing we CAN'T do, however, is to change that object to an array, say. The variable itself cannot be changed.

        For Example:

        /* Not OK */

const obj = {}
obj = []  // Uncaught TypeError: Assignment to constant variable

const arr = [];
arr = ['hi']; // Uncaught TypeError: Assignment to constant variable



/* This is fine */

const obj = {
  hi: 'hello'
}

obj.hi = "bye";
console.log(obj.hi)  // Prints 'bye'


const arr = [];
arr[0] = 'hi';

console.log(arr) // Prints ['hi'];
  5. Difference btw Arrow Functions and reg functions

    • Arrow Functions

      • They're always anonymous (not named). You always need to assign them to a named variable (usually a 'const' variable.

        Because they're anonymous, they don't give us great stack traces where there's an error.

        const multiplyES6 = (x, y) => { return x * y };

      • 'This' is inherited from whatever the parent scope of 'this' is. If there is no parent scope for 'this', it will be Window.

        For example, if we wanted 'this' to be a DOM element that the User clicks, we should use a regular function when adding our eventListener. If we used a fat arrow function, 'this' would not be bound to that DOM element.

        const box = document.querySelector('.box');

box.addEventListener('click', function(){
	this.classList.toggle('opening');  // 'this' is the element that's clicked
	

	// Here, because we used a regular function, 'this' 
	// is scoped to the containing function. Therefor, 'this'
	// is Window, since 'this' is not explicitly defined 
	// inside of the new function.
	setTimeout(function(){
		this.classList.toggle('open');
	});


	// However, if we use an arrow function, 'this'
	// is inherited by it's nearest parent element. 
	// Because 'this' is the DOM element in the parent 
	function, that's the definition in the new function.

	setTimeout(() => {
		this.classList.toggle('open')
	});
});

const convertNumToBinary = (num) => {
	let arr = [];

	function pushBinaryIntoArray(num) {
		if (num % 2 === 0) {
			arr.push(0);
		} else {
			arr.push(1);
		}

		num = Math.floor(num / 2);

		if (num >= 1){
			pushBinaryIntoArray(num);
		}
	}

	pushBinaryIntoArray(num);
	arr = arr.reverse();

	return arr;
}

const findLongestGap = (binaryArray) => {
	let longestGap = 0;
	let currentGap = 0;
	let opening1Registered = false;

	binaryArray.forEach(function(val, index){
		if (val === 0) {
			if (opening1Registered) {
				currentGap++;
			}
		} else {
			if (!opening1Registered){
				opening1Registered = true;
			} else {
				if (currentGap > longestGap) {
					longestGap = currentGap;
					currentGap = 0;
				}
			}
		}
	})

	return longestGap;
}
